# 🔍 Render Debugging Guide

## 🎯 Finding the Root Cause

The error "Database error. Please try again." means something is failing in the database query. With the latest update, the error message will now show **the actual error** instead of a generic message.

---

## 📋 Steps to Debug on Render

### 1. Check Render Logs

After deploying, go to Render dashboard:
1. Click on your service
2. Go to "Logs" tab
3. Look for these messages during startup:

**Expected Success Output:**
```
============================================================
🚀 Starting PhishGuard Application
============================================================

📂 Checking database setup...
   Database directory: /opt/render/project/data
   Database file: /opt/render/project/data/phishguard.db

📊 Database location: /opt/render/project/data/phishguard.db
✅ Database directory is writable

🔧 Initializing database...
✅ Admin password reset to: admin123

✅ Database initialization complete!
```

**If you see all ✅ marks, database is working!**

### 2. Check for Error Messages

If startup fails, logs will show:
```
❌ STARTUP ERROR!
Error: [actual error message]
Database location: /opt/render/project/data/phishguard.db
Directory exists: True/False
Directory writable: True/False
```

Common errors:
- "Read-only file system" → Disk not mounted
- "Permission denied" → Wrong permissions
- "No such file or directory" → DATABASE_DIR not set

### 3. Try Login and Check Logs

After successful startup:
1. Try to login
2. Check logs for login attempt messages:

**Expected:**
```
🔍 Login attempt for: admin
   User found: admin, is_admin: 1
   ✅ Password verified!
   ➡️  Redirecting to: admin panel
```

**If error:**
```
❌ Login error: [actual error message]
[Full stack trace]
```

The actual error message will tell you exactly what's wrong!

## 🔧 Common Issues & Fixes

### Issue 1: "No such table: users"

**Cause:** Database not initialized

**Check:**
```
# In Render logs, search for:
"Initializing database"
"Admin password reset"
```

**Fix:**
- Ensure render.yaml has correct start command
- Check if init_db() is being called
- Verify DATABASE_DIR environment variable is set

### Issue 2: "Read-only file system"

**Cause:** Trying to write to wrong directory

**Check render.yaml:**
```yaml
disk:
  name: phishguard-data
  mountPath: /opt/render/project/data
  sizeGB: 1
```

**Check environment variable:**
```
DATABASE_DIR=/opt/render/project/data
```

**Both must match!**

### Issue 3: "Database is locked"

**Cause:** Multiple processes trying to access database

**Fix:**
- Ensure only one instance is running
- Check if manual deploy triggered while one is running
- Wait 30 seconds and try again

### Issue 4: User not found

**Cause:** Admin user wasn't created

**Check logs for:**
```
✅ Admin password reset to: admin123
```

**If not present:**
- Database initialization failed
- Check earlier in logs for errors

## 🧪 Local Testing

Before deploying to Render, test locally:

### Test 1: Run Test Script
```bash
python test_database.py
```

Should show:
```
✅ ALL TESTS PASSED!
```

### Test 2: Clean Start
```bash
rm -f *.db
python app.py
```

Should show:
```
✅ Database initialization complete!
🌐 Starting web server...
```

### Test 3: Login
```
Go to: http://localhost:5000/login
Username: admin
Password: admin123
```

Should redirect to admin panel!

## 🎯 Render Environment Variables

Ensure these are set in Render dashboard:

```
SECRET_KEY=<auto-generated>
DATABASE_DIR=/opt/render/project/data
```

---

## 📊 render.yaml Checklist

Verify your render.yaml has:

```yaml
services:
  - type: web
    name: phishguard
    runtime: python
    plan: free
    buildCommand: pip install -r requirements.txt
    startCommand: gunicorn app:app
    envVars:
      - key: SECRET_KEY
        generateValue: true
      - key: DATABASE_DIR
        value: /opt/render/project/data
    disk:
      name: phishguard-data
      mountPath: /opt/render/project/data
      sizeGB: 1
```

**Critical:** DATABASE_DIR and disk mountPath must match!

## 🔍 Manual Database Check on Render

If you have SSH access (paid plan):

```bash
# Check if database file exists
ls -lh /opt/render/project/data/phishguard.db

# Check tables
sqlite3 /opt/render/project/data/phishguard.db ".tables"

# Check admin user
sqlite3 /opt/render/project/data/phishguard.db \
  "SELECT username, is_admin FROM users WHERE username='admin'"
```

## 📞 Getting Help

When reporting issues, provide:

1. **Render logs** (first 50 lines after deployment)
2. **Error message** shown on login page
3. **render.yaml** contents
4. **Environment variables** (SECRET_KEY can be redacted)

Example good bug report:
```
Issue: Database error on login

Render logs show:
✅ Database initialization complete!
✅ Admin password reset to: admin123

Login error shows:
"Database error: no such table: users"

render.yaml: [paste here]
DATABASE_DIR: /opt/render/project/data
```

2. ✅ Login page loads without errors
3. ✅ Can login with admin/admin123
4. ✅ Redirects to admin panel
5. ✅ Admin panel shows questions
